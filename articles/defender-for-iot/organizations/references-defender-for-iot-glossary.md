# <a name="defender-for-iot-glossary-for-organizations"></a>組織向け Defender for IoT の用語集

この用語集では、Microsoft Defender for IoT プラットフォームに関する重要な用語と概念について簡単に説明しています。 用語集の関連する用語に移動するには、**詳細情報** のリンクをクリックします。 これは、製品ツールをより迅速に学習して使用するために役立ちます。

<a name="glossary-a"></a>

## <a name="a"></a>A

| 項目 | 説明 | 詳細情報 |
|--|--|--|
| **アクセス グループ** | 大規模な組織のユーザー アクセス要件をサポートするには、アクセス グループのルールを作成します。<br /><br />ルールを使用すると、関連する部署、リージョン、サイト、ゾーンの特定のユーザー ロールによる Defender for IoT オンプレミス管理コンソールの表示および構成アクセスを制御できます。<br /><br />たとえば、Active Directory グループのセキュリティ アナリストに対して、西ヨーロッパの自動車関連のデータにアクセスすることは許可するが、アフリカのデータにはアクセスできないようにすることができます。 | **[オンプレミスの管理コンソール](#o)** <br /><br />**[部署](#b)** |
| **アクセス トークン** | アクセス トークンは、Defender for IoT REST API にアクセスするために生成します。 | **[API](#glossary-a)** |
| **アラート イベントの確認** | 検出されたイベントに対するアラートを 1 回非表示にするように、Defender for IoT に指示します。 アラートは、イベントが再度検出されると再度トリガーされます。 | **[アラート](#glossary-a)<br /><br />[アラート イベントの学習](#l)<br /><br />[アラート イベントのミュート](#m)** |
| **Alert** | 認可されたネットワークの動作からの逸脱、ネットワークの異常、または疑わしいネットワーク アクティビティとトラフィックに関して、Defender for IoT エンジンによってトリガーされるメッセージ。 | **[転送ルール](#f)<br /><br />[除外ルール](#e)<br /><br />[システム通知](#s)** |
| **アラート コメント** | セキュリティ アナリストおよび管理者がアラート メッセージに加えるコメント。 たとえば、アラート コメントには、実行する軽減措置に関する指示や、そのイベントに関して連絡する関係者の名前を記述できます。<br /><br />アラートを確認しているユーザーは、イベントの状態、またはアラートを調査するために実行されたステップが最もよく反映されたコメントを選択できます。 | **[アラート](#glossary-a)** |
| **異常エンジン** | 通常とは異なるコンピューター間 (M2M) の通信と動作を検出する Defender for IoT エンジン。 たとえば、このエンジンでは過剰な SMB サインインの試行が検出される場合があります。 これらのイベントが検出されると、異常アラートがトリガーされます。 | **[Defender for IoT エンジン](#d)** |
| **API** | Defender for IoT によって検出されたデータに外部システムからアクセスし、外部 REST API を使用して SSL 接続経由でアクションを実行できるようにします。 | **[アクセス トークン](#glossary-a)** |
| **攻撃ベクトル レポート** | 悪用される可能性があるエンドポイントの脆弱性チェーンをリアルタイムでグラフィカルに表現したもの。<br /><br />レポートを使用すると、攻撃シーケンスにおける軽減アクティビティの効果を評価して判断できます。 たとえば、システムをアップグレードして攻撃チェーンを破ることによって攻撃者のパスが中断されるかどうか、または代替の攻撃パスが残っているかどうかを評価できます。 これにより、修復および軽減のアクティビティが優先度付けされます。 | **[リスク評価レポート](#r)** |

## <a name="b"></a>B

| 項目 | 説明 | 詳細情報 |
|--|--|--|
| **事業単位** | 特定の業界におけるビジネスの論理的な組織。<br /><br />たとえば、ガラス工場とプラスチック工場を持つグローバル企業は、2 つの異なる部署として管理できます。 特定の部署への Defender for IoT ユーザーのアクセスを制御できます。 | **[オンプレミスの管理コンソール](#o)<br /><br />[アクセス グループ](#glossary-a)<br /><br />[サイト](#s)<br /><br />[ゾーン](#z)** |
| **ベースライン** | 承認されたネットワーク トラフィック、プロトコル、コマンド、デバイス。 Defender for IoT では、ネットワークのベースラインからの逸脱が識別されます。 承認されたベースライン トラフィックを表示するには、データマイニング レポートを生成します。 | **[データ マイニング](#d)<br /><br />[学習モード](#l)** |

## <a name="d"></a>D

| 項目 | 説明 | 詳細情報 |
|--|--|--|
| **データ マイニング** | ネットワーク デバイスに関する包括的で詳細なレポートが生成されます。<br /><br />- **SOC インシデント対応**:リアルタイムで報告され、インシデント対応を迅速に行うことができるようにします。 たとえば、パッチ適用を必要とする可能性のあるデバイスをレポートに一覧表示できます。<br /><br />- **フォレンジクス**: 調査レポートの履歴データに基づくレポート。<br /><br />- **IT ネットワークの整合性**:全体的なネットワーク セキュリティの向上に役立つレポート。 たとえば、認証資格情報が弱いデバイスをレポートに一覧表示できます。<br /><br />- **可視性**:すべてのクエリ項目を対象として、ネットワークのすべてのベースライン パラメーターが表示されるレポート。<br /><br />読み取り専用ユーザーが表示できるようにデータマイニング レポートが保存されます。 | **[ベースライン](#b)<br /><br />[レポート](#r)** |
| **Defender for IoT プラットフォーム** | Defender for IoT センサーとオンプレミスの管理コンソールにインストールされている Defender for IoT ソリューション。 | **[センサー](#s)<br /><br />[オンプレミスの管理コンソール](#o)** |
| **インベントリ デバイス** | Defender for IoT では、次のインベントリでデバイスを識別し、一意の単一ネットワーク デバイスとして分類します。
1. スタンドアロンの IT、OT、IoT デバイス (1 枚以上の NIC を搭載)
1. 複数のバックプレーン コンポーネント (すべてのラック、スロット、モジュールを含む) で構成されるデバイス
1. スイッチやルーターなど、ネットワーク インフラストラクチャとして機能するデバイス (複数の NIC を搭載)。 パブリック インターネット IP アドレス、マルチキャスト グループ、ブロードキャスト グループは、インベントリ デバイスとは見なされません。 60 日を超えて非アクティブ状態にあるデバイスは、非アクティブなインベントリ デバイスとして分類されます。| | **デバイス マップ** | Defender for IoT によって検出されたネットワーク デバイスのグラフィカルな表現。 デバイス間の接続と各デバイスに関する情報が表示されます。 マップを使用して次のことを実行します。<br /><br />- 重要なデバイス情報を取得および制御します。<br /><br />- ネットワーク スライスを分析します。<br /><br />- デバイスの詳細と概要をエクスポートします。 |  **[Purdue レイヤー グループ](#p)** | | **デバイス インベントリ - センサー** | デバイス インベントリには、Defender for IoT によって検出された広範なデバイス属性が表示されます。 次のオプションを使用できます。<br /><br />- 表示されている情報をフィルター処理します。<br /><br />- この情報を CSV ファイルにエクスポートします。<br /><br />- Windows レジストリの詳細をインポートします。 |  **[グループ](#g)** <br /><br />**[デバイス インベントリ - オンプレミスの管理コンソール](#d)** | | **デバイス インベントリ - オンプレミスの管理コンソール** | 接続されているセンサーのデバイス情報は、オンプレミスの管理コンソールからデバイス インベントリで表示することができます。 これにより、オンプレミスの管理コンソールのユーザーは、すべてのネットワーク情報を包括的に把握することができます。 |  **[デバイス インベントリ - センサー](#d)<br /><br />[デバイス インベントリ - データ インテグレーター](#d)** | | **デバイス インベントリ - データ インテグレーター** | オンプレミスの管理コンソールのデータ統合機能を使用すると、他のエンタープライズ リソースからの情報でデバイス インベントリのデータを強化できます。 リソースの例としては、CMDB、DNS、ファイアウォール、Web API などがあります。 |  **[デバイス インベントリ - オンプレミスの管理コンソール](#d)**  |

## <a name="e"></a>E

| 項目 | 説明 | 詳細情報 |
|--|--|--|
| **エンジン** | Defender for IoT の自己学習分析エンジンを使用すると、署名を更新したり、ルールを定義したりする必要がなくなります。 これらのエンジンでは、ICS 固有の動作分析とデータ サイエンスを使用し、異常、マルウェア、操作上の問題、プロトコル違反、およびベースライン ネットワーク アクティビティからの逸脱に関して、OT ネットワーク トラフィックが継続的に分析されます。<br /><br />エンジンによって逸脱が検出されると、アラートがトリガーされます。 アラートは、 **[アラート]** 画面または SIEM から表示および管理できます。 | **[アラート](#glossary-a)** |
| **エンタープライズ ビュー** | Defender for IoT センサーがインストールされている部署、サイト、ゾーンを表すグローバル マップ。 悪意に関するアラートや運用に関するアラートなどの地理的な場所が表示されます。 | **[部署](#b)<br /><br />[サイト](#s)<br /><br />[ゾーン](#z)** |
| **イベント タイムライン** | ネットワークで検出された次のようなアクティビティのタイムライン。<br /><br />- アラートがトリガーされた。<br /><br />- ネットワーク イベント (情報)。<br /><br />- サインイン、ユーザーの削除、ユーザーの作成などのユーザー操作、およびミュート、学習、確認などのアラート管理操作。 センサー コンソールで使用できます。 | - |
| **除外ルール** | 時間間隔、デバイスのアドレス、アラート名、または特定のセンサーに基づいてアラート トリガーを無視するように、Defender for IoT に指示します。<br /><br />たとえば、特定のセンサーによって監視されているすべての OT デバイスで 6:30 から 10:15 の間にメンテナンス手順が実行されることがわかっている場合は、このセンサーが事前定義された期間内にアラートを送信しないように指定した除外ルールを設定できます。 | **[アラート](#glossary-a)<br /><br />[アラート イベントのミュート](#m)** |

## <a name="h"></a>H

| 項目 | 説明 | 詳細情報 |
|--|--|--|
| **Horizon オープン開発環境** | 独自のプロトコルやカスタム プロトコル、または標準から逸脱したプロトコルが実行されている IoT および ICS のデバイスをセキュリティで保護します。 Horizon Open Development Environment (ODE) SDK を使用して、定義されたプロトコルに基づいてネットワーク トラフィックをデコードする dissector プラグインを開発します。 Defender for IoT サービスでは、トラフィックが分析され、完全な監視、アラート、レポートが提供されます。<br /><br />Horizon は次のことに使用します。<br /><br />- Defender for IoT プラットフォームのバージョンをアップグレードすることなく可視性と制御を **拡張** する。<br /><br />- 外部プラグインとしてサイト上で開発することで、機密情報をセキュリティで **保護** する。<br /><br />- アラート、イベント、プロトコルのパラメーターのテキストを **ローカライズ** する。<br /><br />詳細については、お客様サポート担当者にお問い合わせください。 | **[プロトコルのサポート](#p)<br /><br />[ローカライゼーション](#l)** |
| **Horizon カスタム アラート** | (Horizon Framework のトラフィック dissector に基づいて) 任意のプロトコルのカスタム アラートをトリガーすることにより、企業内のアラート管理を強化します。<br /><br />これらのアラートを使用して情報を伝達することができます。<br /><br />- 独自の Horizon プラグイン内のプロトコルおよび基になるプロトコルに基づくトラフィックの検出について。<br /><br />- すべてのプロトコル レイヤーのプロトコル フィールドの組み合わせについて。 | **[プロトコルのサポート](#p)** |

| **内部サブネット** | Defender for IoT によって定義されたサブネット構成。 パブリックな範囲を内部範囲として使用する環境などの場合は、すべてのサブネットを内部サブネットとして解決するように、Defender for IoT に指示できます。 サブネットは、マップやさまざまな Defender for IoT レポートに表示されます。 | **[サブネット](#s)** |

## <a name="r"></a>R

| 項目 | 説明 | 詳細情報 |
|--|--|--|
| **リージョン** | グローバルな組織を地理的な地域に論理的に分割したもの。 たとえば、北米、西ヨーロッパ、東ヨーロッパなどがあります。<br /><br />北米には、さまざまな部署の工場があることがあります。 | **[アクセス グループ](#glossary-a)<br /><br />[部署](#b)<br /><br />[オンプレミスの管理コンソール](#o)<br /><br />[サイト](#s)<br /><br />[ゾーン](#z)** |

| **リスク評価レポート** | リスク評価レポートを使用すると、ネットワーク デバイスごとのセキュリティ スコアと全体的なネットワーク セキュリティ スコアを生成することができます。 全体のスコアは、100% のセキュリティに対する割合を表します。 このレポートには、現在のセキュリティ スコアの向上に役立つ軽減策に関する推奨事項が示されます。 | - |

## <a name="s"></a>S

| 項目 | 説明 | 詳細情報 |

| **選択的プローブ** | Defender for IoT では、IT と OT のトラフィックが受動的に検査され、デバイスおよびその属性と動作などに関する関連情報が検出されます。 場合によっては、一部の情報が受動的ネットワーク分析に表示されないことがあります。<br /><br />これが発生した場合は、Defender for IoT の安全で詳細なプローブ ツールを使用して、以前は到達できなかったデバイスに関する重要な情報を検出することができます。 | - |

| **スマート IT 学習** | 学習期間が終了して学習モードが無効になると、Defender for IoT では、DNS や HTTP 要求などの通常の IT アクティビティの結果として、異常に高いレベルのベースラインの変更が検出されることがあります。 このトラフィックによって、不要なポリシー違反アラートとシステム通知がトリガーされることがあります。 これらのアラートと通知を減らすには、スマート IT 学習を有効にすることができます。 | **[学習モード](#l)<br /><br />[ベースライン](#b)** |
